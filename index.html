The whole system, on one page.

Every foundation specimen, component, UI kit and template in Fabric — rendered live, grouped the way the design tooling shows them. Use Theme to flip light/dark across every preview. Season tints this page's background and any component that opts into --season-canvas / --season-accent.

How to use this catalog ›

1 · Get the files

Clone the repository, then move into it:

# clone & enter
git clone https://github.com/Nexxspace-Development/fabric-design-system.git
cd fabric-design-system

2 · Open it — two easy ways

Just double-click index.html. It ships with an embedded snapshot of the catalog, so every tile renders straight off disk — no terminal needed.

Want the Theme / Season toggles to drive the previews too (not just this page)? Open it over a link or a tiny local server instead:

# Python (built in on most machines)
python3 -m http.server 8000

# or Node
npx serve .

Then visit http://localhost:8000/. Served this way the catalog also re-reads the live manifest, so it auto-syncs as the system grows.

3 · Or just share a link

This repo is set up to publish itself with GitHub Pages, so reviewers can open the catalog in one click — no clone, no terminal:

  1. On GitHub: Settings → Pages.
  2. Under Build and deployment, set Source to GitHub Actions.
  3. Push to main — the bundled workflow deploys automatically to https://nexxspace-development.github.io/fabric-design-system/

Served over real HTTP, the toggles drive every preview and the catalog auto-syncs.

4 · Use Fabric in your project

Link the single stylesheet; load the bundle after React only if you need the components:

<!-- tokens + fonts + base -->
<link rel="stylesheet" href="styles.css">

<!-- components (optional) -->
<script src="_ds_bundle.js"></script>

Full spec, tokens and conventions live in README.md.
