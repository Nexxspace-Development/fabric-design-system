// Fabric for finance

Money has its own grammar.

Fabric is a calm, nature-derived system — but finance arrives with conventions older than any design system: red and green mean direction, numbers must line up, and a loss is not an error. This is how we adapt Fabric to honour them without losing its quiet.

Guideline · v0.5.0 Scope: US / EU · consumer-fintech Layer: tokens/finance.css
01 — PRINCIPLE

Two axes of meaning, kept apart

Fabric already gives green and red a job: sage is success (and the brand primary), rust is danger. That is an evaluative axis — good versus bad. Finance overloads the very same hues with a second, unrelated meaning: direction — a value went up or down. A column of losses is not eight errors; it is just an ordinary Tuesday.

The rule that holds the whole system together: direction is never status. We add a separate directional axis and forbid the two from borrowing each other's tokens.

Evaluative — already in Fabric

Did something succeed or fail? Reserved for outcomes and actions — a posted transfer, a declined card, a destructive delete.

Did a number rise or fall? Reserved for signed quantities — P&L, day change, returns. Neutral valence: a loss is expected, not an alarm.

The one rule that matters most

A negative number must never use --danger. --fin-loss is a visibly calmer, warmer red so that “negative” and “error” stay legibly different — even when they sit on the same screen.

02 — COLOR

The directional tokens

Defined in tokens/finance.css as a parallel axis. Chroma is nudged a hair above Fabric's muted status greens and reds so direction scans at a glance in a dense figure — while staying inside the earthen character. Each adapts to Nightfall and the light / dark toggles automatically.

--fin-gainPositive / upsage lineage, lifted off --success

Binding color to data

Drive direction straight off the sign of the value — never hand-assign colors. The helpers accept a class or a data-fin attribute, so a cell can be coloured from one expression:

  • .fin-gain / .fin-loss / .fin-flat — set text colour directly.
  • [data-fin="gain|loss|flat"] — same, attribute-driven for data-fin={v > 0 ? 'gain' : v < 0 ? 'loss' : 'flat'}.
  • .fin-pill[data-fin] — a soft directional badge; always reinforce it with a sign or glyph inside.
Accessibility — colour is never the only channel

Roughly 1 in 12 men cannot reliably separate red from green, and finance is regulated for it. Every directional value carries a sign (+ / −) or a glyph (▲ / ▼) as well. Colour reinforces; it never carries the meaning alone.

03 — TYPOGRAPHY

Figures that line up

Fabric already has the right bones: IBM Plex Mono is --font-data, and monospace buys column alignment for free. The care is in number behaviour, not font choice.

Tabular, lining figures — always

Numbers that share a column must share a width and a baseline. The .fabric-figure utility sets the mono data voice with tabular-nums lining-nums; .fabric-figure-sans does the same for numerals inside Hanken body text.

✓ Tabular lining — IBM Plex Mono
1,240.50 98.07 12,008.16
✕ Old-style proportional — Newsreader
1,240.50 98.07 12,008.16

The serif's old-style figures bob off the baseline and vary in width — lovely in a sentence, disqualifying in a ledger. Never set financial data in Newsreader. Keep display serif for headlines and quotes only.

House number conventions

  • Signed negatives — −1,240.50 with a true minus (U+2212), not a hyphen. The sign is the primary cue. (Zero is unsigned.)
  • Right-align every numeric column; decimals aligned, consistent decimal places per column.
  • Currency symbol flush to the figure, column reading as a clean block.
  • Thousands separators always on; .fabric-figure-slashed for a slashed zero in dense data.
04 — LEDGERS & TABLES

Direction by column, totals by rule

In a statement, the structure already encodes direction: debits sit under Money out, credits under Money in. So the ledger body stays neutral ink — colour would be noise. Reserve the directional axis for the one genuinely signed figure: the net.

Neutral ink in the body · single rule above the total · the signed net is the one directional figure

  • Totals carry a single rule above (.fabric-total-row); a grand / closing total adds a double rule below (.is-grand, drawn from --fin-rule-strong).
  • Running balances are computed, never hand-summed — so the closing figure always reconciles with the last row.
  • Cash flow is not P&L. A purchase or transfer is neutral; only true gains and losses take directional colour.
05 — IN PRACTICE

Do & don't

▲ Do

  • Colour signed P&L, day change and returns with --fin-*.
  • Pair every colour with a sign or ▲▼ glyph.
  • Set all data in IBM Plex Mono, tabular + lining.
  • Right-align numeric columns; rule your totals.
  • Keep status (Completed / Declined) on the evaluative axis.

▼ Don't

  • Render a negative number in --danger.
  • Lean on colour alone to signal direction.
  • Set figures in Newsreader (old-style serif numerals).
  • Colour neutral cash flow — a buy is not a loss.
  • Flip green=up for East-Asian locales without re-tokening.
